2) 50 Itens de Melhoria (hardening do que já existe)

  1. Autenticação está inconsistente: AuthService.signIn não valida senha/identidade de forma real. Integrar Supabase/Auth provider com verificação de token de identidade, não login mock.
  2. Eliminar segredo JWT de fallback em desenvolvimento; padronizar rotação de chaves via KMS e expiração curta com refresh token.
  3. Substituir leitura direta de process.env em múltiplos pontos por config tipada centralizada com schema (Zod/Env validation).
  4. Aplicar RBAC real: decorator de papéis existe, mas enforcement deve ser obrigatório por guard e policy service.
  5. Corrigir multi-tenancy: x-tenant-id não pode ser confiado sem vínculo criptograficamente atrelado ao token do usuário.
  6. Aplicar Row-Level Security por tenant no PostgreSQL para camada de defesa adicional contra vazamento cross-tenant.
  7. Rever middleware de whitelist de IP; hoje está permissivo. Implementar allowlist por organização com CIDR e bypass auditado.
  8. Desacoplar AppModule monolítico em módulos verticais com boundaries explícitas e contratos internos.
  9. Fortalecer tratamento de exceções: padronizar erro de domínio, código interno e correlation-id em 100% dos endpoints.
  10. Adicionar versionamento de API (/v1) e política de depreciação para evitar quebra de clientes.
  11. Habilitar compressão e tuning de keep-alive para reduzir latência e custo de infra.
  12. Implementar health checks profundos (DB, cache, provider IA, fila, webhook endpoints).
  13. Trocar logs ad-hoc por logging estruturado com máscara de PII obrigatória e trilha de auditoria consistente.
  14. Mapear todos os campos de PII e implementar classificação (P0/P1/P2) + políticas de retenção.
  15. Aplicar criptografia em repouso para dados sensíveis (tokens externos, gravações, transcrições, metadados críticos).
  16. Melhorar modelagem de Communication com enums fortes e constraint para evitar estados inválidos.
  17. Adicionar índices compostos para queries frequentes de CRM (status+updatedAt, score+createdAt, orgId+status).
  18. Revisar unique de email por tenant para tratar nulos e múltiplos canais de contato sem colisões indevidas.
  19. Implementar paginação cursor-based em todas as listagens críticas; evitar paginação por offset em bases grandes.
  20. Aplicar idempotência em endpoints financeiros e webhooks para evitar duplicidade de cobrança/evento.
  21. Blindar Stripe webhook com verificação de assinatura + replay protection + janela temporal.
  22. Introduzir saga/outbox para consistência entre transações internas e efeitos externos (billing, mensagens, CRM).
  23. Separar escrita e leitura em casos de alto volume (CQRS leve) para timelines e analytics.
  24. Implementar circuit breaker e retry exponencial com jitter para integrações externas (HubSpot, voz, IA).
  25. Adicionar cache semântico para respostas de IA por tenant/contexto para reduzir custo de tokens.
  26. Revisar RAG: chunking atual simplista; usar chunk semântico, overlap e score threshold configurável.
  27. Adicionar re-ranking de contexto (BM25 + vetor ou cross-encoder) para melhorar precisão das respostas.
  28. Garantir isolamento de embeddings por tenant com testes automáticos de não-vazamento de contexto.
  29. Trocar retorno de embedding zerado sem API key por falha explícita controlada; mock silencioso mascara problemas.
  30. Padronizar contratos DTO de entrada/saída com validação em runtime para cada endpoint público.
  31. Melhorar cobertura de testes no backend (unitário, integração Prisma, contrato HTTP, carga).
  32. Adicionar testes de segurança automatizados (SAST, dependency scan, secret scan, DAST básico).
  33. Implementar rate limit por tenant/plano e não apenas global por IP.
  34. Introduzir quotas dinâmicas por recurso (chamadas de voz, tokens IA, requisições API, armazenamento).
  35. Criar trilha de custos por tenant para mapear margem unitária e detectar clientes deficitários.
  36. Melhorar front-end: store Zustand com tipos incompletos (updateLeadStatus fora da interface) deve ser corrigido.
  37. Eliminar comentários de “placeholder” no front e transformar em fluxos navegáveis reais com router e tracking.
  38. Implementar camada de API client tipado no web, com retry/backoff e tratamento de erros homogêneo.

  41. Habilitar feature flags para rollout progressivo de funcionalidades de alto risco.
